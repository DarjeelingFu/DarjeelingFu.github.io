测试报告-WEEK 8

我们的α迭代成果： Uuchan/SoftwareEngineeringProject - 微信开发者代码管理 (qq.com)

1.version 1.1 bug list

本次测试出现的bug

  • 消息问题：消息页显示的消息不正确。

  • 房间成员问题：当无其他成员加入时，房间详情页成员显示为0但是成员列表有房主，而房间管理页的成员列表不显示房主；有的时候进入房间组员会显示不全；重复点击申请加入会出现成员缺失。

  • 数据审查问题：创建房间的人数为0或负数时也可以创建成功；创建房间的时候可以输入特殊表情后创建成功的房间在房间列表未显示；创建房间时标题过长创建不成功。

  • 跳转问题：修改房间信息后回到修改之前的页面，没有刷新修改信息。

  • 排版问题：房间介绍信息显示溢出 、房间管理页房间介绍信息没有缩进。

  • 加载问题：页面加载较慢。

有待改进的地方

  • 新房间在房间页的最底部，应显示在顶部。
  • 点击“加入房间”按钮后应有提示信息，先由房主审核，审核通过后准许加入；加入房间之后，“加入房间”按钮应改为“退出房间”按钮。
  • 加入房间以后自己应可以申请退出房间。
  • 管理房间时应可以修改房间人数。
  • 后续可以显示一下房间的头像。
  • 房间标题应有字数限制
  • 房间主页内容有待添加。
  • 创建房间成功应该跳转房间详情页。

2.场景测试

场景一

  • 典型用户：房主
  • 需求：想要寻找SRDP项目成员。
  • 场景描述：
    • 扫码打开小程序。
    • 点击创建房间的按钮，填写相关的信息，包括房间的类型（SRDP组队），创建房间的目的，介绍等等。
    • 创建房间后，等待其他人加入房间。一段时间后，进入消息页面检查是否有新的成员。
    • 进入房间详情页面，在成员一栏点击相应成员的头像，查看详细的个人信息。

场景二

  • 典型用户：成员
  • 需求：想要寻找SRDP队伍。
  • 场景描述：
    • 扫码打开小程序。
    • 浏览位于首页的房间列表，选择合适SRDP队伍的房间。
    • 点击房间进入详情页，查看房间的详细信息和当前的房间成员。
    • 发现有合适的房间，点击申请加入，试图加入该房间。
    • 一段时间后，打开消息页面，查看是否有新的的消息显示自己已加入该房间。
